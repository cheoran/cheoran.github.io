## 09.3 NAT
사설 IP를 공인 IP로 변환하는 기술

### SNAT
- 내부 → 외부

### DNAT
- 외부 → 내부 (포트 포워딩)

```text
[10.0.0.5] --(SNAT)--> [1.2.3.4]
```

## 09.12 출판 심화: Stateless vs Stateful
- **Stateless**: 패킷 단위 필터링
- **Stateful**: 연결 상태를 추적

### 차이점
- Stateful은 응답 트래픽을 자동 허용 가능
- Stateless는 룰이 많아지고 관리가 어렵다

## 9.x NAT 심화: PAT(포트 주소 변환)
PAT는 여러 사설 IP를 **하나의 공인 IP + 포트**로 변환한다.

### 예시
- 내부 10.0.0.10:50000 → 공인 1.2.3.4:40001
- 내부 10.0.0.11:50001 → 공인 1.2.3.4:40002

### 실무 포인트
- 대부분 가정/기업 NAT는 PAT 방식
- 포트 충돌 관리가 중요

## 09.x 초보자용 자세한 설명

### 인바운드/아웃바운드란
- **인바운드**: 외부 → 내부로 들어오는 트래픽
- **아웃바운드**: 내부 → 외부로 나가는 트래픽

### 방화벽 규칙 읽는 방법
- 조건: 출발지/목적지/포트/프로토콜
- 행동: 허용(Allow) 또는 차단(Deny)
- 보통 규칙은 위에서 아래로 검사한다.

### NAT가 필요한 이유
- 내부 IP는 인터넷에서 직접 라우팅되지 않는다.
- NAT가 공인 IP로 변환해 외부와 통신 가능하게 한다.

### PAT(포트 변환)
- 하나의 공인 IP에 여러 내부 연결을 **포트로 구분**한다.
- 대부분 가정/회사 라우터가 이 방식이다.

## 초보자 완전 해설

### 인바운드/아웃바운드 한 줄 요약
- 인바운드: 바깥에서 안으로 들어오는 것
- 아웃바운드: 안에서 밖으로 나가는 것

### 방화벽 룰 읽기 쉬운 방법
- "누가(출발지)" → "어디로(목적지)" → "무슨 문(포트)" → "허용/차단"

### NAT가 왜 필요?
- 내부 주소는 인터넷에서 사용 불가
- 공인 주소로 바꿔줘야 나갈 수 있다
