## 04.1 IPv6 주소 구조

![assets/ipv6_header.png](./assets/ipv6_header.webp)

> 그림 04-1. IPv6 헤더 구조

IPv6는 128비트 주소 체계이며 16비트씩 8블록으로 표현한다.

예시:

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

### 단축 표기 규칙

- 연속된 0 블록은 `::`로 축약 가능 (한 번만 사용 가능)
- 앞의 0은 생략 가능

예:

```
2001:db8:85a3::8a2e:370:7334
```

:::note[IPv6 필요 이유]

- IPv4 주소는 부족
  :::

## 04.2 주소 유형

- **유니캐스트**: 하나의 대상
- **멀티캐스트**: 여러 대상
- **애니캐스트**: 가장 가까운 대상

### 주요 주소 범위(자주 보는 것만)

- **글로벌 유니캐스트**: `2000::/3` (일반적인 공인 주소)
- **링크 로컬**: `fe80::/10` (같은 링크 내부 통신)
- **고유 로컬(ULA)**: `fc00::/7` (사설 주소 개념)

## 04.3 듀얼스택

IPv4와 IPv6를 함께 사용하는 방식. 점진적 전환에 사용된다.

### 전환 방식 개요

- 듀얼스택: IPv4/IPv6 동시 운영
- 터널링: IPv4 망 위에 IPv6 캡슐화
- NAT64/DNS64: IPv6에서 IPv4 서비스 접근

### 기본 라우팅 개념

- 기본 경로는 `::/0`로 표현한다.
- 기본 게이트웨이는 보통 RA(라우터 광고)로 전달된다.

## 04.4 기본 실습

## 04.7 심화 이론: NDP(Neighbor Discovery)

- ARP 대신 **NDP**를 사용
- ICMPv6 기반으로 동작
- 주요 메시지: NS/NA, RS/RA

## 04.9 문제 + 모범답안

1. **문제**: `fe80::/10`은 어떤 용도인가?
   **답**: 링크 로컬 주소 범위로, 같은 링크 내부 통신에 사용된다.
2. **문제**: IPv6에서 ARP 역할을 대체하는 것은?
