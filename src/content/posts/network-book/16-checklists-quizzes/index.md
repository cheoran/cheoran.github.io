## 16.3 모범답안
1) 10.0.0.0/24의 사용 가능 호스트 수: 254
2) ARP 요청 브로드캐스트 이유: 같은 네트워크 내 모든 장치에 질의해야 함
3) TCP 3-way: SYN → SYN/ACK → ACK
4) NAT 필요 이유: 사설 IP를 공인 IP로 변환
5) L4/L7 차이: L4는 포트 기반, L7은 HTTP 기반
6) CNAME 역할: 별칭 제공
7) DHCP DORA: Discover → Offer → Request → Ack

---

## 16.4 용어 사전 (확장)
- **캡슐화**: 계층을 내려가며 헤더를 붙이는 과정
- **역캡슐화**: 수신 측에서 헤더를 벗기는 과정
- **MTU**: 한 번에 전송 가능한 최대 패킷 크기
- **RTT**: 왕복 시간
- **SYN/ACK**: TCP 연결 수립 신호
- **TTL**: 패킷 수명(라우터 경유 시 감소)
- **NAT**: 사설 IP ↔ 공인 IP 변환
- **SNAT/DNAT**: NAT 유형
- **세션 고정**: 동일 사용자 요청을 같은 서버로 전달
- **헬스체크**: 백엔드 상태 확인
