:::note[섹션 개요]

- 운영 점검 체크리스트와 퀴즈로 학습 상태를 점검한다.
- 보안/모니터링/백업/자동화를 빠르게 훑는다.
- 시험 대비·실무 셀프체크용으로 활용한다.
  :::

## 데일리 체크리스트

- [ ] 서비스 상태 점검
- [ ] 디스크/메모리 사용량 확인
- [ ] 최근 로그 오류 여부 확인
- [ ] 변경 사항 기록

---

## 운영 점검 체크리스트 (클라우드 엔지니어)

- [ ] SSH 하드닝: 패스워드 로그인 차단, root 접속 금지, fail2ban 동작
- [ ] 방화벽/SG: 리슨 포트와 규칙 일치 여부
- [ ] 패치/커널 업데이트 일정 확인
- [ ] 백업: 최근 스냅숏/LVM 스냅/RAID 상태 확인
- [ ] 모니터링 알람: 디스크 80%↑, 메모리 압력, TCP CLOSE_WAIT, 높은 load
- [ ] 로그 수집: 중앙 전송/회전 정상 동작
- [ ] 컨테이너: 이미지 서명/취약점 스캔, seccomp/AppArmor 적용
- [ ] 자동화: Ansible 인벤토리 최신, 크리덴셜 로테이션

---

## 퀴즈 (기초)

1. `ip route`의 기본 게이트웨이는 어디에서 확인되는가?
2. systemd 서비스 로그는 어떤 명령으로 확인하는가?
3. `chmod 644`의 의미를 설명하라.
4. `ss -lntup`로 확인 가능한 정보는?

---

## 퀴즈 (심화)

1. 기본 방화벽 정책을 설명하라.
2. fstab 항목 중 자동 마운트 실패 시 부팅에 영향 주지 않는 옵션은?
3. cron이 실행되지 않을 때 가장 먼저 볼 것 3가지는?
4. MTU/MSS를 조정해야 하는 상황 2가지를 말하라.
5. LVM 스냅숏과 RAID1 리빌드의 차이를 설명하라.
6. seccomp/AppArmor의 역할을 한 줄씩 설명하라.
