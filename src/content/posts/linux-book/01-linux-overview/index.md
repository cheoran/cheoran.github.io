## 핵심 개념

**커널**
하드웨어(CPU, 메모리, 디스크)와 프로그램 사이를 연결하는 핵심 부분이다. 우리가 파일을 열거나 네트워크를 쓰는 동작은 결국 커널이 처리한다. 그래서 리눅스를 이해할 때 가장 먼저 떠올려야 하는 게 커널이다.

**쉘**
명령을 입력하는 창이다. 우리가 터미널에 치는 모든 명령은 쉘이 받아서 실행한다. 예를 들어 `bash`, `zsh` 같은 종류가 있고, 어떤 쉘이든 “명령을 해석하는 역할”은 같다.

**유틸리티**
`ls`, `cp`, `ip` 같은 기본 도구 모음이다. 리눅스는 “작은 명령들을 조합해서 문제를 푼다”는 철학이 있어서 유틸리티의 역할이 크다.

**배포판**
같은 리눅스 커널 위에 “패키지 구성과 기본 설정”을 묶어 만든 제품군이다. Ubuntu, Debian, Rocky, Fedora 등은 커널은 비슷하지만 기본 패키지/정책이 다르다.

**배포판 계열 구분**
가장 쉬운 구분법은 패키지 명령이다. Ubuntu/Debian은 `apt`, RHEL/Rocky는 `dnf`(또는 `yum`), SUSE는 `zypper`를 사용한다. 결국 “커널은 비슷하고 관리 도구가 다르다”는 점을 기억하면 된다.

---

## 필수 명령어

- `uname -a`, `cat /etc/os-release` : 커널/배포판 확인
- `whoami`, `id`, `hostname` : 사용자/호스트 정보 확인
- `man`, `--help`, `tldr` : 도움말 보는 습관

---

## 실습 1: 기본 정보 확인

```bash
lin> uname -a
lin> cat /etc/os-release
lin> whoami
lin> hostname
```

여기서 **커널 버전**과 **배포판 이름**을 찾을 수 있어야 한다.

---

## 실습 2: 도움말 탐색

```bash
lin> man ip
lin> ip --help
```

명령어는 외울 필요 없다. “도움말을 보는 방법”을 익히는 게 더 중요하다.

## 트러블슈팅

- `man`이 안 열리면: `sudo apt install man-db`
- `tldr`가 없다면: `sudo apt install tldr`
