## 한줄 요약

리눅스의 “폴더 지도”를 익혀서 필요한 파일을 빠르게 찾는다.

## 핵심 경로

**/etc**
설정 파일이 모여 있는 곳이다. 서비스가 왜 안 뜨는지 확인할 때 가장 먼저 보는 폴더다.

**/var/log**
로그가 저장되는 곳이다. 장애 원인을 찾을 때 이 경로를 가장 많이 확인한다.

**/var/lib**
서비스가 상태/데이터를 저장하는 곳이다. DB 데이터, 캐시, 서비스 메타정보가 들어간다.

**/usr/bin**
일반 사용자가 실행하는 명령어가 들어 있다.

**/bin, /sbin**
시스템이 꼭 필요한 기본 명령(관리자 도구 포함)이 들어 있다. 부팅/복구 시에도 사용된다.

**/home**
사용자 홈 디렉터리. 개인 파일과 설정이 저장된다.

왜 중요한가?
리눅스는 폴더 역할이 정형화되어 있어 “어디를 보면 되는지”를 아는 것만으로 문제 해결 속도가 빨라진다.

---

## 마운트란?

- 디스크나 파일시스템을 특정 폴더에 “붙여서” 쓰는 것
- 예: `/data` 폴더에 다른 디스크를 붙여 사용
- 마운트하지 않으면 디스크가 있어도 접근할 수 없다

---

## 필수 명령어

- `ls -al`, `pwd`, `cd` : 위치/목록 확인
- `df -h`, `du -sh`, `lsblk` : 디스크 사용량 확인
- `mount`, `umount` : 마운트/언마운트

---

## 실습 1: 디렉터리 구조 탐색

```bash
lin> ls -al /
lin> ls -al /etc
lin> ls -al /var/log
```

이 명령을 사용하는 이유
- 루트(`/`) 아래에 어떤 폴더가 있는지 보고, 설정/로그 위치를 익힌다.

예상 결과(예시):
```text
bin  boot  dev  etc  home  lib  lib64  usr  var
```

해석:
- 여기서 `/etc`, `/var/log`, `/home` 같은 핵심 경로를 직접 확인한다.

---

## 실습 2: 디스크 사용량 확인

```bash
lin> df -h
```

이 명령을 사용하는 이유
- 전체 디스크 사용량을 확인한다. “디스크 꽉 참” 문제를 진단할 때 가장 먼저 본다.

예상 결과(예시):
```text
Filesystem      Size  Used Avail Use% Mounted on
/dev/sda1        50G   18G   30G  38% /
```

```bash
lin> du -sh /var/log
```

이 명령을 사용하는 이유
- 특정 폴더의 실제 크기를 확인한다. 로그가 비정상적으로 커졌는지 확인 가능하다.

예상 결과(예시):
```text
420M    /var/log
```

```bash
lin> lsblk
```

이 명령을 사용하는 이유
- 디스크/파티션 구조를 확인한다. 새 디스크가 붙었는지 확인할 때 사용한다.

예상 결과(예시):
```text
NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda      8:0    0   50G  0 disk
└─sda1   8:1    0   50G  0 part /
```
